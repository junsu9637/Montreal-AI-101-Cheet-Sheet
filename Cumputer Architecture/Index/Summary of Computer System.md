## 컴퓨터 시스템

```markdown
사용자가 원하는 작업 내용을 포함하는 소프트웨어와 작업을 수행하기 위한 하드웨어로 구성된 시스템
```

### 컴퓨터 시스템의 개념적 구성

컴퓨터 시스템은 **하드웨어**와 **소프트웨어** 그리고 **펌웨어**로 구성

**하드웨어**
```markdown
시스템을 구성하는 물리적 부품으로 이루어진 전자/기계적 장치
```

**소프트웨어**
```markdown
1. 하드웨어가 작업을 수행할 순서와 방법을 지시하는 명령어로 구성된 프로그램
2. 프로그램이 수행에 필요한 절차, 규칙, 관련 문서
```

**펌웨어** <하드웨어와 소프트웨어의 중간>
```markdown
하드웨어를 제어하는 데 필수적인 프로그램을 하드웨어로 만든 것
고정도가 높음
기본적인 내장 프로그램으로 ROM에 내장 > 효율 증가
```

### 컴퓨터 시스템의 4대 기능


**입력**
```markdown
외부 세계에서 내부 세계로 정보 받아들임
키보드
```

**처리**
```markdown
입력된 정보를 중앙처리장치를 통해 산술연산 or 논리연산을 수행하여 정보를 가공
```

**저장**
```markdown
입력된 정보나 처리된 정보를 저장장 저장
하드디스크
```

**출력**
```markdown
입력 or 처리 or 저장된 정보를 출력장치를 통해 내부 세계에서 외부 세계로 내보냄
```

### 비트, 바이트, 워드


**비트**
```markdown
정보를 구성하는 최소 단위 (0 or 1)
```

**바이트**
```markdown
정보 표현의 기본 단위
1byte = 8bit
```

*과거 4bit 크기인 니블을 정보 표현의 기본 단위로 사용한 적 도 있음*

**워드**
```markdown
중앙처리장치에서 한 번에 처리할 수 있는 비트의 집합
컴퓨터 마다 워드의 크기가 다름
보편적으로 1word = 4byte = 32bit
```

*word는 bit나 byte와는 다르게 의미를 가진 정보이다. 즉 word 단위 정보는 제어장치가 **명렁어**로 해석하고, 연산장치가 수치와 같은 **데이터**로 취급한다.*


## 컴퓨터 하드웨어

```markdown
1. 중앙처리장치
2. 기억장치
3. 주변장치
```

### 중앙처리장치

*일반적으로 CPU와 마이크로프로세서를 구분없이 사용하지만 다른 용어*
> CPU = 컴퓨터 시스템의 논리적 구성 요소          
  마이크로프로세서 = CPU를 집적회로 형태로 만든 물리적 구성 요소

**CPU**
```markdown
컴퓨터 시스템에 부착된 모든 장치의 동작을 제어하고 명령을 실행하는 장치
CPU = 제어장치 + 연산장치 + 레지스터 + 내부 버스
```

**제어장치**
```markdown
메인 메모리로부터 적절한 순서로 명령어를 인출 및 해독하여 컴퓨터 시스템에 연결된 장치에 제어신호 할당
```

**연산장치**
```markdown
제어 신호에 따라 명령어대로 데이터를 처리하는 장치
제어장치가 지시하는 순서에 따라 데이터에 대한 산술연산이나 논리연산을 수행
```

**레지스터**
```markdown
데이터를 처리하는 동안 사용할 값이나 연산의 중간 결과를 일시적으로 저장
고속 기억장치
```

**CPU 내부버스**
```markdown
CPU 내부 구성 요소를 연결하는 배선 집합
```

### 기억장치

```markdown
컴퓨터 시스템에 의해 처리될 프로그램과 데이터를 저장
```

#### 메모리 계층구조
`레지스터 > 캐시 메모리 > 메인 메모리 > 보조기억장치` 

### 주번장치

**입력장치**
```markdown
컴퓨터에서 처리할 데이터를 외부에서 입력하기 위한 장치
```

**출력장치**
```markdown
컴퓨터에서 처리한 데이터를 외부로 보내기 위한 장치
```

**저장장치**
```markdown
영구적으로 데이터를 저장하는 장치
입출력의 복합적인 장치이기 때문에 주변장치로 구분
```

### 기타 하드웨어

**버스**
```markdown
컴퓨터 시스템의 기능 장치를 연결하는 배선의 집합
```

**시스템 버스**
> 데이터 버스
>> CPU-메인 메모리 or 주변장치 사이 데이터를 전송       
   데이터버스 구성 배선 수 = CPU가 한번에 전송 가능한 bit = word

> 주소버스
>> CPU가 시스템 구성요소를 식별하기 위한 주소 정보를 전송하는 배선         
   주소버스 배선 수 = CPU가 접속할 수 있는 최대 메모리 용량

> 제어버스
>> CPU가 시스템 구성 요소의 동작을 제어하는 데 사용되는 배선

**메인 보드**
```markdown
1. 컴퓨터의 실행 환경 조성
2. 설정 정보 유지
3. 컴퓨터의 안정적 구동 보장
4. 모든 장치의 데이터 입출력 교환 보장
```

*기능 장치 = 컴퓨터 성능, 메인보드 = 기능장치 사이 호환성 확장의 범위*

**메인보드 구성**
```markdown
1. 칩셋
2. 확장카드 슬롯
3. 저장장치 포트
4. 주 전원 포트
5. 보조전원 포트
6. CPU 소켓
7. 메모리 소켓
```

*칩셋은 다양한 역할을 수행하기 때문에 방열판이나 냉각판을 추가한다.*

*기존 메인보드는 노스브리지와 사우스브리지를 칩셋으로 사용*

*노스브리지 = 고속장치 구동 및 메모리 컨트롤러 포함*

*사우스브리지 = 저속장치 구동 및 I/O 컨트롤러 포함*

*최근에는 노스브리지를 CPU로 옮겨 사우스브리지만 사용하거나 둘을 하나로 합침*
